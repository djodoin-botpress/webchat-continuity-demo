Webchat User Continuity — Node demo

Log in with a username. The backend ensures a Botpress user exists for it (calling POST /:clientId/users) and returns the userKey JWT. The same username on any device → same Botpress user → same conversation history.

How it works

1. POST /api/login { username }
2. Server looks up the username in .data/users.json
3. If missing → POST {WEBCHAT_API}/:clientId/users → save { userId, userKey }
4. Server returns { userId, userKey, conversationId } to this page
5. Page loads the inject script and calls window.botpress.init({ clientId })
6. On 'webchat:initialized', page calls
   window.botpress.updateUser({ userKey }) — explicit, supported association
